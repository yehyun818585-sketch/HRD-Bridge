# 정보 미러링 대시보드 구현 계획

## 프로젝트 개요
공동훈련센터용 정보 미러링 대시보드 - 기업 진행 현황을 읽기 전용으로 보고, 댓글로 피드백 가능

## 핵심 요구사항
1. **정보 가시성** - 기업별 과정 진행 현황 한눈에 파악
2. **읽기 전용** - 공동훈련센터는 수정 불가
3. **댓글 기능** - 서류 추가/수정 요청 피드백

## 기술 스택
- **Frontend**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **Backend/DB**: Supabase (PostgreSQL + RLS + Realtime) - 새로 생성
- **인증**: Supabase Auth + OAuth (구글/카카오)

## 구현 단계

### Step 1: 프로젝트 초기 설정
- Next.js 프로젝트 생성 (TypeScript + Tailwind)
- Supabase 프로젝트 새로 생성 (supabase.com)
- Supabase 클라이언트 설정 (@supabase/supabase-js)
- 기본 레이아웃 구성

### Step 1.5: OAuth 설정
- Supabase에서 Google OAuth 활성화
- Supabase에서 Kakao OAuth 활성화
- 로그인/로그아웃 UI 구현

### Step 2: 데이터베이스 스키마
```sql
-- 기업 테이블
companies (id, name, created_at)

-- 과정 테이블
courses (id, company_id, name, status, stage, issues, created_at)

-- 댓글 테이블
comments (id, course_id, user_id, content, created_at)

-- 사용자 테이블 (역할 포함)
profiles (id, email, role, name) -- role: 'company' | 'center'
```

### Step 3: RLS 정책 (권한 분리)
- 센터: courses/companies 조회만 가능
- 센터: comments 작성/조회 가능
- 기업: 자신의 courses 수정 가능

### Step 4: 페이지 구성
- `/` - 대시보드 홈
- `/companies` - 기업 리스트 (센터용)
- `/companies/[id]` - 기업 상세 + 댓글

### Step 5: 실시간 댓글 알림
- Supabase Realtime 구독
- 새 댓글 시 토스트 알림
