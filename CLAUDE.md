# HRD-Bridge

HRD(인적자원개발) 지원사업 참여 기업들이 제출해야 하는 필수서류(사업계획서, 전담인력등록 등)의 제출 현황과 이슈를 센터-기업 간에 실시간으로 공유하는 정보 미러링 대시보드.

## 기술 스택

- **프레임워크**: Next.js 16 (App Router), React 19, TypeScript
- **스타일**: Tailwind CSS 4
- **백엔드/DB**: Supabase (`@supabase/ssr`, `@supabase/supabase-js`) — 인증, 실시간 구독(`postgres_changes`), 파일 메타데이터 저장
- **주요 엔티티**: `companies` → `courses` → `course_files`(업로드 파일), 역할 기반(센터/기업) 인증 및 댓글(`CommentSection`, `CompanyCommentSection`)

## 필수서류 검증 철학

이 프로젝트에서 "서류가 정상 제출됐다"를 판별하는 코드는 아래 원칙을 따른다. 자세한 내용은 [.claude/skills/document-validation/SKILL.md](.claude/skills/document-validation/SKILL.md) 참고.

1. **파일명 매칭은 1차 분류일 뿐**이며 확정 판단 근거로 쓰지 않는다.
2. **문서 내용(텍스트)** 을 추출해 필수 섹션이 실제 본문에 존재하는지 2차로 확인한다.
3. **기준 서식(스키마)** 과 대조해 필수 항목 누락 여부를 3차로 점검한다.
4. 검증 결과는 항상 **확인됨 / 누락 / 불일치** 3단계로만 리포트한다.
5. **AI/시스템이 자동 승인·자동 판단하지 않는다.** 최종 판단은 항상 사람이 하며, 시스템은 대조 결과만 제시한다.

1단계(파일명 매칭)는 `src/lib/document-validation.ts`의 `getPdfFiles`/`checkStaffDuplication`(순수 함수, 클라이언트에서도 import 가능)에 있고, 2·3단계(문서 내용 추출·기준 서식 대조)는 `src/lib/document-validation-server.ts`의 `validateDocumentContent`(`server-only`, PDF 파싱 포함)에 있다. 브라우저(클라이언트 컴포넌트)는 `/api/validate-document`를 통해 2·3단계 결과를 받는다.

## 작업 방식

### 파일 접근
- 로컬 경로의 파일 **확인(읽기)**을 지시하면, 되묻지 말고 해당 경로를 직접 읽고 진행한다.
- 단, 파일 **수정·삭제·명령 실행**은 기존대로 사용자 확인을 받는다.

### 코드 수정
- 코드를 고치기 전에 **먼저 관련 파일을 읽어 현재 구조를 파악**한 뒤 수정한다.
- 큰 변경은 바로 실행하지 말고, **무엇을 왜 바꿀지 먼저 설명**하고 승인받는다.
- 요청한 것만 수정하고, **시키지 않은 리팩터링·파일 생성은 하지 않는다.**
- 한 번에 하나의 작업 단위로 처리하고, 여러 파일을 동시에 뒤엎지 않는다.
사용자가 sql을 실행해야 할 경우, 실행할 코드를 화면에 보여준다.  

### 검증·확인
- 수정 후에는 **변경한 부분이 실제로 동작하는지** 확인 방법(빌드, 실행)을 함께 알려준다.
- 확실하지 않은 부분은 추측하지 말고 **모른다고 말하거나 확인 방법을 제시**한다.

### 소통
- 답변은 **간결하게**, 핵심 위주로 한다.
- 코드에 대한 설명은 한국어로 한다.
